## 方案一：GitHub Actions 自动打包（推荐⭐⭐⭐）

**特点**：无需安装任何开发环境，一切在云端完成。最适合普通用户。

**预估时间**：30分钟（其中您操作5分钟，等待25分钟）

---

### 第一步：准备项目文件（5分钟）

#### 1.1 解压项目

```bash
# Windows: 右键 ZIP 文件 → 解压到当前文件夹
# Mac/Linux:
unzip signalis_android_trpg_v2.zip -d signalis_trpg/
cd signalis_trpg/signalis_android
```

解压后目录结构应为：

```
signalis_android/
├── main.py              # 主程序（114KB）
├── signalis.kv          # 样式文件（5.4KB）
├── buildozer.spec       # 打包配置（2.2KB）
├── README.md            # 说明文档
├── .github/
│   └── workflows/
│       └── build_apk.yml    # GitHub Actions配置（自动打包脚本）
├── bin/                 # 空目录（APK将输出到这里）
└── .buildozer/          # 构建缓存目录
```

#### 1.2 初始化 Git 仓库

```bash
# 确保在 signalis_android 目录内
cd signalis_android

# 初始化 git 仓库
git init

# 添加所有文件
git add .

# 提交
git commit -m "SIGNALIS TRPG v2.0 - 初始版本"
```

---

### 第二步：创建 GitHub 仓库（5分钟）

#### 方法一：使用 GitHub CLI（命令行，推荐）

```bash
# 1. 安装 GitHub CLI（如果未安装）
# Windows: winget install --id GitHub.cli
# Mac: brew install gh
# Linux: 
#   curl -fsSL https://cli.github.com/packages/githubcli-archive-keyring.gpg | sudo dd of=/usr/share/keyrings/githubcli-archive-keyring.gpg
#   sudo chmod go+r /usr/share/keyrings/githubcli-archive-keyring.gpg
#   echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" | sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null
#   sudo apt update && sudo apt install gh

# 2. 登录 GitHub
ght auth login
# 按提示选择：
#   - What account do you want to log into? → GitHub.com
#   - What is your preferred protocol for Git operations on GitHub? → HTTPS
#   - Authenticate Git with your GitHub credentials? → Yes
#   - How would you like to authenticate? → Login with a web browser
#   - 复制给出的8位验证码，粘贴到浏览器打开的页面中

# 3. 创建仓库并推送
ght repo create signalis-trpg --public --source=. --remote=origin --push
# 输出示例：
# ✓ Created repository YourName/signalis-trpg on GitHub
# ✓ Added remote https://github.com/YourName/signalis-trpg.git
# ✓ Pushed commits to https://github.com/YourName/signalis-trpg.git
```

#### 方法二：使用 GitHub 网页（图形界面）

1. 打开浏览器，访问 https://github.com/new
2. **Repository name** 填写：`signalis-trpg`
3. **Description** 填写：`SIGNALIS TRPG 数值判定程序 - 安卓APK`
4. 选择 **Public**（公开仓库）
5. 不要勾选 "Add a README file" 或 "Add .gitignore"
6. 点击 **Create repository**
7. 复制页面上显示的以下命令并执行：

```bash
# 在 signalis_android 目录内执行
git remote add origin https://github.com/你的用户名/signalis-trpg.git
git branch -M main
git push -u origin main
# 提示输入用户名和密码/Token时，输入你的GitHub用户名和Personal Access Token
```

### 第三步：触发自动打包（1分钟）

#### 3.1 进入 Actions 页面

1. 浏览器打开：`https://github.com/你的用户名/signalis-trpg/actions`
2. 或者在你的仓库页面点击顶部 **Actions** 标签

#### 3.2 运行工作流

1. 在左侧边栏找到 **"Build SIGNALIS TRPG APK"**
2. 点击它展开
3. 点击右侧的 **"Run workflow"** 按钮（灰色下拉按钮）
4. 再次点击绿色的 **"Run workflow"** 确认

```
┌─────────────────────────────────────────┐
│  Build SIGNALIS TRPG APK                │
│                                         │
│  ┌──────────────────────────────┐      │
│  │  ▼ Run workflow              │      │
│  │                              │      │
│  │  Use workflow from           │      │
│  │  ┌────────────────────────┐  │      │
│  │  │ Branch: main ▼         │  │      │
│  │  └────────────────────────┘  │      │
│  │                              │      │
│  │  ┌──────────────────────┐    │      │
│  │  │  Run workflow        │    │      │
│  │  └──────────────────────┘    │      │
│  └──────────────────────────────┘      │
└─────────────────────────────────────────┘
```

#### 3.3 等待编译完成

- 点击刚刚启动的工作流任务
- 你会看到实时日志，显示以下步骤：

```
▶ Set up job
▶ Checkout code
▶ Set up Python
▶ Install dependencies          ← 安装Java、Buildozer等（约5分钟）
▶ Cache Android SDK/NDK         ← 首次会下载（约15分钟）
▶ Build APK with Buildozer      ← 编译APK（约10分钟）
  ↳ 下载Android SDK
  ↳ 下载Android NDK
  ↳ 编译Python for Android
  ↳ 编译Kivy
  ↳ 打包APK
▶ Upload APK artifact           ← 上传APK文件
▶ Create Release                ← 创建发布版本
```

**总耗时**：首次约25-35分钟（需要下载SDK/NDK），后续约15分钟（使用缓存）。

**状态颜色说明**：
- 🟡 黄色 = 进行中
- 🟢 绿色 = 成功
- 🔴 红色 = 失败

---

### 第四步：下载 APK（1分钟）

#### 方式 A：从 Actions 页面下载

1. 工作流完成后（全部显示绿色✓），点击 **"Upload APK artifact"** 步骤
2. 或者在页面底部找到 **Artifacts** 区域
3. 点击 **"signalis-trpg-apk"** 下载 ZIP 文件
4. 解压 ZIP，得到 `signalis-2.0-arm64-v8a_armeabi-v7a-debug.apk`

#### 方式 B：从 Releases 页面下载（推荐）

1. 访问 `https://github.com/你的用户名/signalis-trpg/releases`
2. 找到最新的 Release（如 `v1`）
3. 在 Assets 区域点击 APK 文件直接下载

## 方案二：本地 Buildozer 打包

**特点**：在自己的电脑上完成全部编译，完全掌控。适合开发者。

**系统要求**：
- Ubuntu 20.04+ / Debian 11+（推荐，成功率最高）
- 或 WSL2 (Windows Subsystem for Linux)
- 或 Docker
- **磁盘空间**：至少 10GB 空闲空间
- **内存**：至少 4GB RAM
- **网络**：稳定的互联网连接（需下载约 2GB 的SDK/NDK）

---

### 第一步：安装系统依赖（5分钟）

#### 1.1 Ubuntu / Debian

```bash
# 更新包列表
sudo apt-get update

# 安装构建工具
sudo apt-get install -y \
    autoconf \
    automake \
    libtool \
    pkg-config \
    zlib1g-dev \
    libncurses5-dev \
    libncursesw5-dev \
    libffi-dev \
    libssl-dev \
    libltdl-dev \
    git \
    wget \
    unzip \
    default-jdk \
    python3 \
    python3-pip \
    python3-venv

# 验证Java安装
javac -version
# 应输出类似：javac 17.0.x
```

#### 1.2 Windows (使用 WSL2)

```powershell
# 1. 以管理员身份打开 PowerShell，安装 WSL2
wsl --install -d Ubuntu
# 重启电脑

# 2. 打开 Ubuntu 终端，执行上面的 Ubuntu 安装命令

# 3. 项目文件放在 Windows 的 D盘，在 WSL 中访问：
cd /mnt/d/signalis_android  # 假设项目在D盘
```

#### 1.3 Mac (使用 Docker，因为Buildozer不支持macOS原生)

```bash
# 1. 安装 Docker Desktop
#    下载地址：https://www.docker.com/products/docker-desktop

# 2. 使用 Docker 运行 Ubuntu 容器
docker run -it --name buildozer \
    -v "$(pwd)/signalis_android:/home/buildozer/build" \
    ubuntu:22.04 bash

# 3. 在容器内执行 Ubuntu 安装命令（见1.1节）
```

---

### 第二步：安装 Buildozer（3分钟）

```bash
# 安装 Python 依赖
pip3 install --user buildozer cython

# 添加到 PATH
export PATH="$HOME/.local/bin:$PATH"

# 验证安装
buildozer --version
# 应输出：Buildozer 1.x.x

# 将 PATH 添加到 ~/.bashrc（永久生效）
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
```

---

### 第三步：准备项目（2分钟）

```bash
# 解压项目
unzip signalis_android_trpg_v2.zip
cd signalis_android

# 验证文件
ls -la
# 应看到：main.py  signalis.kv  buildozer.spec  README.md

# 设置JAVA_HOME（如果 javac 不在默认路径）
export JAVA_HOME=/usr/lib/jvm/default-java
export PATH=$JAVA_HOME/bin:$PATH
```

---

### 第四步：开始打包（首次25-35分钟）

```bash
# 在 signalis_android 目录内执行
buildozer android debug
```

#### 打包过程详解

Buildozer 会自动执行以下步骤：

**阶段1：环境准备（约5分钟）**
```

[INFO]: Check configuration tokens
[INFO]: Preparing build
[INFO]: Check requirements for android
[INFO]: Search for Git (git)  -> found at /usr/bin/git
[INFO]: Search for Cython (cython)  -> found at /home/xxx/.local/bin/cython
[INFO]: Search for Java compiler (javac)  -> found at /usr/bin/javac
[INFO]: Search for Java keytool  -> found at /usr/lib/jvm/.../bin/keytool
```

**阶段2：下载 Android SDK/NDK（约15分钟，仅首次）**
```
[INFO]: Android SDK is missing, downloading...
[INFO]: Android NDK is missing, downloading...
[INFO]: Extracting Android SDK...
[INFO]: Extracting Android NDK...
[INFO]: Installing platform api 33...
[INFO]: Installing build tools...
```

**阶段3：编译 Python for Android（约5分钟）**
```
[INFO]: Building hostpython3 for armeabi-v7a/arm64-v8a
[INFO]: Building python3 for armeabi-v7a/arm64-v8a
[INFO]: Building sdl2 for armeabi-v7a/arm64-v8a
[INFO]: Building kivy for armeabi-v7a/arm64-v8a
```

**阶段4：打包 APK（约3分钟）**
```
[INFO]: Building application... (this may take a while)
[INFO]: Packaging...

```

---

### 第五步：获取 APK

```bash
# APK 文件位于 bin/ 目录
ls -lh bin/*.apk

# 输出示例：
# -rw-r--r-- 1 user user 15M Jun 17 12:00 bin/signalis-2.0-arm64-v8a_armeabi-v7a-debug.apk

# 如果连接了安卓手机，可以直接安装：
buildozer android deploy run
# 或使用 adb：
adb install bin/signalis-2.0-arm64-v8a_armeabi-v7a-debug.apk
```

---

### 第六步：后续打包（使用缓存，约10分钟）

首次打包后，SDK/NDK 会被缓存到 `~/.buildozer/` 目录。后续打包只需：

```bash
# 修改代码后重新打包（快很多！）
cd signalis_android
buildozer android debug
```

如果要清理缓存重新下载：
```bash
buildozer android clean
buildozer android debug
```

## 常见问题排查

### ❌ 问题1：javac not found

```
[ERROR]: Java compiler (javac) not found, please install it.
```

**解决**：
```bash
sudo apt-get install -y default-jdk
# 验证
javac -version
```

### ❌ 问题2：下载SDK/NDK超时

```
[ERROR]: Failed to download Android SDK
```

**解决**：设置镜像源
```bash
# 编辑 buildozer.spec
# 找到 android.sdk_path 和 android.ndk_path
# 或手动下载后放入 ~/.buildozer/android/platform/
```

### ❌ 问题3：内存不足

```
[ERROR]: Out of memory
```

**解决**：增加交换空间
```bash
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

### ❌ 问题4：GitHub Actions 工作流未触发

**解决**：
1. 确保 `.github/workflows/build_apk.yml` 文件在仓库根目录
2. 检查 Actions 是否启用：仓库 Settings → Actions → General → 选择 "Allow all actions and reusable workflows"
3. 确保使用的是 main 或 master 分支

### ❌ 问题5：GitHub Actions 编译失败

**解决**：
1. 点击失败的 workflow
2. 查看具体失败的步骤（红色❌）
3. 查看日志中的错误信息
4. 常见原因：
   - 文件编码问题 → 确保 main.py 是 UTF-8 编码
   - 缺少文件 → 确保 signalis.kv 存在
   - 缓存损坏 → 在Actions页面点击 "Clear caches and re-run"

## 安装APK到手机

### 方法一：直接传输安装

1. 将 APK 文件传输到手机：
   - USB 数据线连接电脑
   - 蓝牙传输
   - 微信/QQ/邮件发送到手机
   - 上传到网盘后手机下载

2. 在手机上打开 APK 文件

3. 系统会提示 "允许安装未知来源应用"，点击 **设置 → 允许**

4. 点击 **安装**

### 方法二：使用ADB命令安装

```bash
# 连接手机，开启USB调试（设置 → 开发者选项 → USB调试）
adb devices
# 应显示设备列表

# 安装APK
adb install -r bin/signalis-2.0-arm64-v8a_armeabi-v7a-debug.apk
# -r 表示覆盖安装（如果已安装）

# 启动应用
adb shell am start -n org.signalis.signalis_trpg/org.kivy.android.PythonActivity
```

### 首次运行注意事项

1. **横屏模式**：应用会自动横屏显示。如果未自动横屏，请关闭手机的"自动旋转锁定"
2. **分辨率适配**：应用会自动适配您的手机屏幕分辨率
3. **字体大小**：如果文字过小，请在手机设置中调整"显示大小"为默认或较小

## 附：buildozer.spec 关键配置说明

```ini
[app]
title = SIGNALIS TRPG Adjudicator          # 应用名称
package.name = signalis_trpg               # 包名
package.domain = org.signalis              # 域名
source.dir = .                             # 源代码目录
source.include_exts = py,kv,json           # 包含的文件类型
version = 2.0.0                            # 版本号
requirements = python3,kivy                # Python依赖
orientation = landscape                    # 横屏模式 ← 关键！
fullscreen = 1                             # 全屏显示

[buildozer]
log_level = 2                              # 日志详细程度（2=INFO）
warn_on_root = 0                           # 不警告root用户

[android]
android.api = 33                           # Android API级别（Android 13）
android.minapi = 21                        # 最低API（Android 5.0）
android.sdk = 33
android.ndk = 25b
android.archs = arm64-v8a, armeabi-v7a     # 支持64位和32位
android.allow_backup = True
android.permission.INTERNET = True
android.permission.WRITE_EXTERNAL_STORAGE = True
android.permission.READ_EXTERNAL_STORAGE = True
```
