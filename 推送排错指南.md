# GitHub 推送排错指南：仓库为空

## 问题诊断清单

请按顺序执行以下检查命令，找到问题所在：

### 检查1：远程仓库是否已关联

```bash
git remote -v
```

**正确输出**：
```
origin  https://github.com/512Penrose/signalis-trpg.git (fetch)
origin  https://github.com/512Penrose/signalis-trpg.git (push)
```

**如果无输出**：说明未关联远程仓库，执行：
```bash
git remote add origin https://github.com/512Penrose/signalis-trpg.git
```

---

### 检查2：本地分支是否正确

```bash
git branch
```

**正确输出**：
```
* main
```

**如果输出 `* master`**：需要重命名分支：
```bash
git branch -M main
```

---

### 检查3：提交历史是否存在

```bash
git log --oneline
```

**正确输出**（至少有一行）：
```
f80d563 SIGNALIS TRPG v2.0
```

**如果无输出**：说明没有提交，执行：
```bash
git add .
git commit -m "SIGNALIS TRPG v2.0"
```

---

### 检查4：GitHub仓库是否为空

1. 浏览器访问 `https://github.com/512Penrose/signalis-trpg`
2. 如果看到 "Quick setup" 页面（有HTTPS/SSH等选项）→ **仓库为空，正常**
3. 如果看到 "This branch is even with master/main" → **仓库有内容，不正常**

---

## 常见原因与解决方案

### 原因1：创建GitHub仓库时勾选了README（最常见！）

**症状**：
- GitHub仓库页面显示 "1 commit" 或 "Initial commit"
- 推送时报错：`rejected: non-fast-forward`

**解决**：强制推送（用本地内容覆盖远程）

```bash
git push -u origin main --force
```

> ⚠️ 这会清空GitHub上的初始README，用本地文件替代

---

### 原因2：分支名不匹配

**症状**：推送 `main` 但本地是 `master`

**解决**：
```bash
git branch -M main
git push -u origin main
```

或者直接用master推送：
```bash
git push -u origin master
```

---

### 原因3：Token权限不足或认证失败

**症状**：
- 推送时提示 `fatal: Authentication failed`
- 或提示 `403 Forbidden`

**解决**：

1. 创建新的Personal Access Token：
   - 访问 https://github.com/settings/tokens/new
   - Note: `signalis-trpg-push`
   - Expiration: `No expiration`（或自定义日期）
   - 勾选权限：
     - [x] `repo`（完整仓库控制）
   - 点击 **Generate token**
   - **立即复制Token**（页面关闭后无法再次查看！）

2. 使用Token推送：
```bash
git push -u origin main
# Username: 512Penrose
# Password: ghp_xxxxxxxxxxxxxxxxxxxx（粘贴你的Token，不显示字符）
```

---

### 原因4：使用了GitHub密码而非Token

**症状**：
- 2021年后GitHub不再支持密码验证
- 提示 `Support for password authentication was removed`

**解决**：必须使用Personal Access Token代替密码（见原因3）

---

### 原因5：远程URL错误

**症状**：`Could not resolve host: github.com`

**解决**：
```bash
# 检查当前远程URL
git remote -v

# 如果错误，先删除再重新添加
git remote remove origin
git remote add origin https://github.com/512Penrose/signalis-trpg.git
```

---

### 原因6：GitHub Actions工作流被禁用

**症状**：代码推送成功，但Actions页面空白

**解决**：
1. 访问 `https://github.com/512Penrose/signalis-trpg/settings/actions`
2. 选择 **Allow all actions and reusable workflows**
3. 点击 **Save**
4. 刷新Actions页面
